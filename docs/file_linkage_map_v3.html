Our Triton forward and backward kernels execute block-level sparse operations directly inside GPU SRAM. By loading query and key blocks only when their tree coordinates match, we skip high-bandwidth memory (HBM) retrieval entirely.

Theoretical Core

1. From p-adic Distance to Differentiable Trees

In the algebraic geometry backend, the $p$-adic distance matrix is computed using prime valuations ($v_p$). The distance decreases as the power of the prime dividing the coordinate differences increases:

d_p(x, y) = p^{-v_p(x - y)}

In the transformer, this is replaced by the differentiable expectation of tree distance based on path matching probabilities across tree nodes:

expected_dist = levels - sum(cumprod(agreement_probs))

2. Adèlic Product Formula & Multi-Prime Routing

The adèlic product formula connects all prime places to reconstruct global algebraic spaces:

A_Q = R x \prod_p Q_p

The V2/V3 research modules implement this by grouping the attention heads and assigning co-prime arities (e.g. $p=2$ and $p=3$ concurrently). The intersection of their local tree structures covers the sequence, replicating the global properties of adèlic numbers.
